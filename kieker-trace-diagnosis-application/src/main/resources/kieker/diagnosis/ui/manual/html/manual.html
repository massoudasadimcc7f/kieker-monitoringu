Introduction

Kieker Trace Diagnosis is a tool that has been developed to tabularly prepare Kieker monitoring logs and make them searchable. A focus has been put on memory usage and performance, which means that bigger logs in the range of several million method calls can still be opened.

Requirements and Installation

The following requirements are necessary to use Kieker Trace Diagnosis.
  • Windows or Linux
  • Java Runtime Environment 8 Update 60 or higher
The usage of Oracle's JDK for the tool is recommended. The troubleshooting for the usage of OpenJDK can be found in section OpenJDK. The memory usage of the tool depends on the size of the monitoring logs. As guideline it is recommended to provide twice the size of the monitoring logs.

For the installation, it is only necessary to unpack the archive Kieker Trace Diagnosis-${version}-linux.tgz, respectively Kieker Trace Diagnosis-${version}-windows.zip, into an abritary folder. The application can subsequently be started with the star script start.sh, respectively start.bat, in the folder bin. The application should have write permissions for the folder from which it has been started.

Views

Traces

This tab shows the single traces of the monitoring logs. Incomplete traces are not shown here. This view allows to search for various parameters. Per default the filter is only applied on the top method call, which is the root node of the trace. If necessary, the search can be widened to the whole trace by using the checkbox "Search in the whole trace". If the checkbox "Search with regular expressions" is active, the filter masks "host", "class", "method", and "exceptions" can be filled with regular expressions with the default Java syntax.

Method Calls

This tab shows the single method calls of the monitoring logs. Ignored methods and methods of incomplete traces are not shown. This view allows to search for various parameters. If the checkbox "Search with regular expressions" is active, the filter masks "host", "class", "method", and "exceptions" can be filled with regular expressions with the default Java syntax.

Aggregated Method Calls

This tab shows the method calls of the monitoring logs in aggregated form. Methods with identical host, class, method, and exception are combined to a single aggregated method call. Ignored methods and methods of incomplete traces are not shown. This view allows to search for various parameters. If the checkbox "Search with regular expressions" is active, the filter masks "host", "class", "method", and "exceptions" can be filled with regular expressions with the default Java syntax.

Statistics

This tab shows various statistics for the imported monitoring logs and the application. The current (heap) memory usage of the application can be seen here as well.

Settings

The settings allow to configure the tool.

The point "Appearance" allows to perform settings which has direct influence on the visual representation of the data. For instance, the time unit or the representation of method calls can be changed. The checkbox "Show non-monitored time" shows the time in the trace which has been used in the method call, but cannot be break down in detail due to missing monitoring.

With the help of the point "Aggregate method calls in traces" method calls can be combined in traces. This is useful if a trace has several thousand method calls, but, for instance, only the slowest are important. It can be choosen from various aggregation types.

Monitoring Settings

The monitoring settings can be used to enable and configure the monitoring for Kieker Trace Diagnosis. Generally speaking this is only useful for the development of Kieker Trace Diagnosis or for teaching purposes. The logs from this monitoring can be opened with the tool as well.

Troubleshooting

Usage of OpenJDK

The usage of OpenJDK instead of Oracle's JDK is basically possible. However, it can be necessary to install OpenJFX explicitly.

Unable to create file logs/Kieker Trace Diagnosis.log

If this error message appears during the start, it is very likely that Kieker Trace Diagnosis does not have write permissions in the current working directory. In this case the application can not write any log files, but can be used as usual.

OutOfMemoryError

If an OutOfMemoryError is raised while using the tool, the monitoring logs are to big to be loaded by Kieker Trace Diagnosis. In this case the parameter -Xmx in the start script should be set to a higher value.
